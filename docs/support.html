LH2GPX – Support

Lokaler Standortverlauf-Import, Analyse und Export

Was ist LH2GPX?

LH2GPX importiert Ihren Google-Standortverlauf (und weitere Formate) lokal auf Ihrem Gerät, analysiert die Daten und ermöglicht den Export in gängige GPS-Formate. Alle Verarbeitungen finden vollständig lokal statt – ohne Kontozwang, ohne Cloud-Pflicht.

Häufige Fragen

Werden meine Daten hochgeladen?

Nein. Die App verarbeitet alle Daten lokal auf Ihrem Gerät. Daten verlassen das Gerät nur, wenn Sie selbst aktiv einen Export auslösen oder – als optionale Funktion – einen eigenen Live-Aufzeichnungs-Server konfigurieren. Der Entwickler empfängt keine Daten.

Brauche ich ein Konto oder eine Registrierung?

Nein. LH2GPX benötigt kein Konto, keine E-Mail-Adresse und keine Anmeldung.

Gibt es Tracking, Werbung oder Analytics?

Nein. Die App enthält kein Tracking, keine Werbenetzwerke, keine Crashreporter und keine Analysedienste. Die App-Store-Datenschutzangabe lautet „Keine Daten erfasst".

Welche Importformate werden unterstützt?

  • Google Standortverlauf – app_export.json und .zip-Export
  • GPX 1.1 (Tracks und Wegpunkte)
  • TCX 2.0 (Trainings-/Aktivitätsdateien)

Welche Exportformate werden unterstützt?

  • GPX – GPS Exchange Format
  • KML – Keyhole Markup Language (Google Earth)
  • KMZ – komprimiertes KML
  • GeoJSON
  • CSV – tabellarisch

Was ist die optionale Live-Aufzeichnung?

LH2GPX kann optional aktuelle GPS-Koordinaten an einen Server Ihrer Wahl übertragen. Diese Funktion ist standardmäßig vollständig deaktiviert und erfordert, dass Sie selbst eine eigene Server-URL eintragen. Der Entwickler betreibt keinen Empfangsserver. Die Aktivierung liegt ausschließlich bei Ihnen.

Meine Datei wird nicht erkannt – was tun?

Stellen Sie sicher, dass die Datei im unterstützten Format vorliegt (JSON/ZIP von Google Takeout, GPX oder TCX). Falls das Problem weiterhin besteht, melden Sie es bitte über GitHub Issues mit einer anonymisierten Beschreibung des Problems.

Die App stürzt ab oder zeigt einen Fehler – wie melde ich das?

Bitte öffnen Sie ein GitHub Issue mit einer Beschreibung des Fehlers, der iOS-Version und der App-Version. Keine privaten Standort- oder Nutzerdaten im Issue teilen.
